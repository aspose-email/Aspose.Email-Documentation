## **Introducción**
### **Apache POI - HSMF**
**HSMF** es la implementación pura en Java del formato Outlook MSG del proyecto POI.

En este momento, proporciona acceso de lectura de bajo nivel a todo el archivo, junto con una forma accesible para el usuario de acceder al contenido textual común de los archivos MSG.

Lee más en [poi.apache.org](http://poi.apache.org/hsmf/index.html)
### **Aspose.Email para Java**
Aspose.Email para Java es una biblioteca de clases de Java que permite a las aplicaciones Java leer y escribir archivos de mensajes de correo electrónico en varios formatos sin Microsoft Outlook.

Proporciona clases para leer y actualizar archivos MSG, EML, EMLX, OFT, agregar/quitar archivos adjuntos y destinatarios, actualizar el asunto, el cuerpo y otras propiedades de los archivos MSG.

Lee más en [aspose.com](http://www.aspose.com/java/email-component.aspx)
### **Aspose.Email Java para Apache POI - HSMF**
El proyecto Aspose.Email para Apache POI - HSMF muestra cómo se pueden realizar diferentes tareas utilizando las API de Aspose.Email Java en comparación con Apache POI HSMF. El proyecto también cubre las características que solo están disponibles en las API de Aspose.Email pero no en Apache POI HSMF para tratar mensajes de correo electrónico.

Este proyecto es útil para desarrolladores que desean comparar Apache POI HSMF con Aspose.Email O migrar de Apache POI HSMF a Aspose.Email.
## **Requisitos del Sistema y Plataformas Soportadas**
### **Requisitos del Sistema**
**Los siguientes son los requisitos del sistema para ejecutar Aspose.Email Java para Apache POI - HSMF:**

- Java 1.4 o superior instalado.
- Componente Aspose.Email descargado.
- Bibliotecas Apache POI - HSMF descargadas.
## **Plataformas Soportadas**
**Las siguientes son las plataformas soportadas:**

- Apache POI 3.11.
- Aspose.Email 4.2.0.0 y superior.
- IDE de Java (Eclipse, NetBeans, IntelliJ, ...).

### **Cómo configurar el código fuente**
Por favor, sigue estos pasos simples para abrir y extender el código fuente mientras usas:
#### **Eclipse IDE**
1. Descarga/Clona el código fuente.
1. Abre Eclipse y elige **Archivo** > **Importar...**
1. Selecciona **General** > **Proyectos Existentes en el Espacio de Trabajo**.
1. Navega para **seleccionar el directorio raíz** del último código fuente que has descargado.
1. Haz clic en **Finalizar** para importar exitosamente el proyecto descargado.
#### **NetBeans IDE**
1. Descarga/Clona el código fuente.
1. Abre NetBeans y elige **Archivo** > **Nuevo Proyecto...**
1. En la pestaña **Elegir Proyecto**: Selecciona **Java** > **Proyecto Java con Fuentes Existentes**.
1. En la pestaña **Nombre y Ubicación**: Proporciona tu **Nombre de Proyecto** y **Carpeta de Proyecto** deseados.
1. En la pestaña **Fuentes Existentes**: Haz clic en **Agregar Carpeta...** y navega para seleccionar el directorio raíz del último código fuente que has descargado.
1. Haz clic en **Finalizar** para crear exitosamente el proyecto descargado.

Para configurar **Aspose.Email para Java API**, incluye la URL del repositorio de Aspose y la dependencia de maven de Aspose.Email para Java en tu **pom.xml** como sigue:
#### **Repositorio Maven de Aspose**
``` java

 <repositories>

    <repository>

        <id>aspose-maven-repository</id>

        <name>Repositorio Maven de Aspose</name>

        <url>https://releases.aspose.com/java/repo/</url>

    </repository>

</repositories>

```
#### **Dependencia Maven de Aspose.Email para Java**
``` java

 <dependency>

    <groupId>com.aspose</groupId>

    <artifactId>aspose-email</artifactId>

    <version>5.5.0</version>

    <classifier>jdk16</classifier>

</dependency>

```

**Nota:** Por favor, utiliza la versión más reciente disponible de la API de Aspose.
